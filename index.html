Configuration Modules

Select a configuration module below to filter visual guides and commands.

01

Device Identity

Naming your router or switch for easy identification

1

Enter Privileged Mode

Elevates permissions from User Exec (>) to Privileged Exec (#).

Required
Cisco IOS Terminal
Switch> enable

Tip: You can also use en as a shortcut for this command.

2

Enter Global Config

Access mode to change device-wide settings. Shortcut: conf t

Required
Cisco IOS Terminal
Switch# configure terminal

Tip: Use conf t as a shortcut for this command.

3

Set Hostname

Updates the prompt to your custom name instantly. Choose a descriptive name for easy identification.

Config
Cisco IOS Terminal
Switch(config)# hostname R1-Main
R1-Main(config)# ‚Üê Hostname changed!

Tip: Shortcut: host [name]. Use descriptive names like NYC-RTR-01.

02

User Mode Security

Securing console access to the device

1

Select Console Port

Enter line configuration mode for the physical console port (port 0).

Config
Cisco IOS Terminal
R1(config)# line console 0

Tip: Use line con 0 as a shortcut for this command.

2

Set Password

Define the case-sensitive password string for console access.

Security
Cisco IOS Terminal
R1(config-line)# password cisco123

Tip: Shortcut: pass [your-password]. Use strong passwords in production!

3

Enable Login Check

Critical: Forces the router to ask for the password on console access.

Critical
Cisco IOS Terminal
R1(config-line)# login

Important: Without login, the password won't be enforced! This step is mandatory.

03

Privileged Mode Security

Protecting the 'enable' command with encryption

1

Set Encrypted Secret

Uses MD5 encryption. This is the safest and recommended way to protect privileged mode access.

Encrypted
Cisco IOS Terminal
R1(config)# enable secret class123

Tip: Always use enable secret instead of enable password for MD5 encryption!

04

Password Encryption

Encrypt plain text passwords in configuration

1

Enable Password Encryption

Applies weak encryption (Type 7) to all current and future plain text passwords (like console line password) to prevent shoulder surfing.

Security
Cisco IOS Terminal
R1(config)# service password-encryption

Note: This encryption is easily crackable but good for preventing casual viewing. Use enable secret for real security.

2

Disable Password Encryption

Stops encrypting new passwords. Already encrypted passwords remain encrypted.

Undo
Cisco IOS Terminal
R1(config)# no service password-encryption

Note: Previously encrypted passwords will NOT be decrypted. Only new passwords will be stored in plain text.

05

Final Verification

Exit configuration mode and verify your settings

1

Exit Config Mode

Drop back to Privileged Exec mode (#) instantly. This exits all sub-configuration modes.

Exit
Cisco IOS Terminal
R1(config)# end
R1# ‚Üê Back to privileged mode

Tip: You can also use Ctrl+Z as a keyboard shortcut to exit configuration mode instantly.

02

VLAN Creation

Define and name your Layer 2 segments

1

Create VLAN ID

Enter global configuration mode and create the VLAN.

Configure
Cisco IOS Terminal
Switch(config)# vlan 10
2

Name VLAN

Assign a descriptive name to the VLAN for easier identification.

Cisco IOS Terminal
Switch(config-vlan)# name SALES
03

Port Assignment

Assign interfaces to VLANs

1

Select Interfaces

Choose a single port (fa0/6) or a range (range fa0/1-5).

Cisco IOS Terminal
Switch(config)# interface fa0/6 // Single Port
Switch(config)# interface range fa0/1 - 5 // Multiple Ports
2

Set Access Mode

Configure ports as access ports (connecting to end devices).

Cisco IOS Terminal
Switch(config-if-range)# switchport mode access
3

Assign VLAN

Assign the selected ports to VLAN 10.

Cisco IOS Terminal
Switch(config-if-range)# switchport access vlan 10

Tip: Use interface range to configure multiple ports at once.

04

VLAN Shutdown

Administratively disable VLAN interfaces

1

Shutdown VLAN Interface

Administratively shutdown a VLAN to disable all traffic on it without deleting it.

Management
Cisco IOS Terminal
Switch# configure terminal
Switch(config)# interface vlan 10
Switch(config-if)# shutdown
%LINK-5-CHANGED: Interface Vlan10, changed state to administratively down
2

Verify Shutdown Status

Check if the VLAN interface is administratively down.

Verify
Cisco IOS Terminal
Switch# show ip interface brief
Interface              IP-Address      OK? Method Status                Protocol
Vlan1                  unassigned      YES unset  up                    up
Vlan10                 unassigned      YES unset  administratively down down
3

Restore VLAN (No Shutdown)

Re-enable the VLAN interface if it was previously shut down.

Restore
Cisco IOS Terminal
Switch# configure terminal
Switch(config)# interface vlan 10
Switch(config-if)# no shutdown
%LINK-3-UPDOWN: Interface Vlan10, changed state to up

Tip: Use show ip interface brief again to verify the VLAN is back up.

01

Device Identity

Naming your router or switch for easy identification

1

Enter Privileged Mode

Elevates permissions from User Exec (>) to Privileged Exec (#).

Required
Cisco IOS Terminal
Switch> enable

Tip: You can also use en as a shortcut for this command.

2

Enter Global Config

Access mode to change device-wide settings. Shortcut: conf t

Required
Cisco IOS Terminal
Switch# configure terminal

Tip: Use conf t as a shortcut for this command.

3

Set Hostname

Updates the prompt to your custom name instantly. Choose a descriptive name for easy identification.

Config
Cisco IOS Terminal
Switch(config)# hostname R1-Main
R1-Main(config)# Hostname changed!

Tip: Shortcut: host [name]. Use descriptive names like NYC-RTR-01.

02

Console Security

Locking down physical access to the device

1

Select Console Port

Enter line configuration mode for the physical console port (port 0).

Config
Cisco IOS Terminal
R1(config)# line console 0

Tip: Use line con 0 as a shortcut for this command.

2

Set Password

Define the case-sensitive password string for console access.

Security
Cisco IOS Terminal
R1(config-line)# password cisco123

Tip: Shortcut: pass [your-password]. Use strong passwords in production!

3

Enable Login Check

Critical: Forces the router to ask for the password on console access.

Critical
Cisco IOS Terminal
R1(config-line)# login

Important: Without login, the password won't be enforced! This step is mandatory.

03

Privileged Mode Security

Protecting the 'enable' command with encryption

1

Set Encrypted Secret

Uses MD5 encryption. This is the safest and recommended way to protect privileged mode access.

Encrypted
Cisco IOS Terminal
R1(config)# enable secret class123

Tip: Always use enable secret instead of enable password for MD5 encryption!

04

Verification & Saving

Check your work and save configuration to NVRAM

1

Exit Config Mode

Drop back to Privileged Exec mode (#) instantly. This exits all sub-configuration modes.

Exit
Cisco IOS Terminal
R1(config)# end
R1# Back to privileged mode

Tip: You can also use Ctrl+Z as a keyboard shortcut to exit configuration mode instantly.

2

Save to NVRAM

Important: Ensures your configuration survives a reboot by copying to startup-config.

Critical
Cisco IOS Terminal
R1# copy running-config startup-config
Destination filename [startup-config]? <Enter>
Building configuration...
[OK]

Tip: Shortcuts: copy run start or wr (write memory).
